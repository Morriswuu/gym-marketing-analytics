# Gym Member Cardio Analysis 🏃‍♂️💦

本報告針對健身房有氧運動 (Cardio) 群組的平均卡路里低於警戒值 (900 大卡) 的現象進行分析。包含資料庫 SQL 查詢、Python 統計運算分析以及後續的營運行銷建議。

## 1. 資料庫分析 SQL 

這支 SQL 負責從系統原始的 BigQuery 資料庫抓出 Cardio 族群，並依照「經驗等級」與「性別」交叉計算人數與平均消耗卡路里：

```sql
SELECT 
    Experience_Level, 
    Gender, 
    COUNT(*) as user_count,
    AVG(Calories_Burned) as avg_calories_burned
FROM `gym-project-491908.gymmember.gym`
WHERE Workout_Type = 'Cardio'
GROUP BY Experience_Level, Gender
ORDER BY avg_calories_burned ASC
```

## 2. Python 量化分析與視覺化

完整的分析腳本以及生成圖表的程式碼，請參見本資料夾下的 `cardio_analysis.py`。

**重點量化結論（以 885 大卡總平均為 100% 基準）：**

*   **落後群體 (拉低平均主力，佔整體過半)**
    *   **等級 1 女性 (59人)**：683.5 大卡 ➡️ **達標率 77.2%** *(落後 22.8%)*
    *   **等級 1 男性 (50人)**：732.7 大卡 ➡️ **達標率 82.8%** *(落後 17.2%)*
    *   **等級 2 女性 (45人)**：867.6 大卡 ➡️ **達標率 98.0%** *(微幅落後 2.0%)*
*   **領先群體 (拉高平均主力)**
    *   等級 2 男性 (57人)：978.7 大卡 ➡️ 達標率 110.6%
    *   等級 3 女性 (22人)：1173.4 大卡 ➡️ 達標率 132.6% 
    *   等級 3 男性 (22人)：1270.2 大卡 ➡️ 達標率 143.5%

可以看到，**有高達 60% 的學員因為難以達到 900 大卡的熱量消耗，導致整體平均被拉下。**

## 3. 行銷與營運策略決策 (Actionable Insights)

針對這些族群特性的分析，我們提出三大營運優化建議：

1. **針對「新手入門 (Level 1)」— 破冰與遊戲化 (Gamification)**:
   由於初學者人數最多且熱量消耗最低，可結合 App 推出「Cardio 破冰徽章挑戰」。降低門檻例如單場完成 800 大卡即給予鼓勵，利用溫和升級法逐步延長他們的運動耐力，避免挫折停跑。
   
2. **針對「中階瓶頸 (Level 2 女性)」— 推廣進階課程 (Upsell/Cross-sell)**:
   該群組消耗落在 867.6 大卡，距離警戒線只差一點點，代表已經有基礎習慣但遇到強度瓶頸！可以精準發放「HIIT 燃脂飛輪課免費體驗券」或「進階專屬 PT 諮詢」，來提升客單價並助其突破關卡。
   
3. **App 即時推播機制**:
   會員開始有氧後，若器材偵測耗能來到 800-850 區間，APP 可以給予即時鼓勵通知：「太棒了！只差 50 大卡就能突破今日標準，堅持下去！」以促使邊緣使用者達標。
